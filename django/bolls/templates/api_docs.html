Bible API Documentation

RESTful API for accessing Bible translations, verses, and search functionality

Base URL: https://api.prayerpulse.io
Production Ready

Introduction

The PrayerPulse Bible API provides programmatic access to multiple Bible translations, search functionality, and user features. All endpoints return JSON responses with CORS enabled for cross-origin requests.

Response Format: All responses are in JSON format with UTF-8 encoding.

Authentication

Most read endpoints do not require authentication. User-specific features (bookmarks, history, notes) require authentication via Django session or OAuth.

Public Endpoints: No authentication required for verse retrieval, search, and book metadata endpoints.

Quick Start

Get started with a simple request to fetch John 3:16:

Request
curl https://api.prayerpulse.io/get-verse/YLT/43/3/16/
Response
{
  "pk": 26133,
  "verse": 16,
  "text": "for God did so love the world, that His Son -- the only begotten -- He gave..."
}

Verses

Retrieve Bible verses, chapters, and random verses.

GET
/get-verse/{translation}/{book}/{chapter}/{verse}/
Retrieve a single Bible verse by translation, book, chapter, and verse number.
Path Parameters
translation string
Translation code (e.g., YLT, KJV, ASV)
book integer
Book number (1-66)
chapter integer
Chapter number
verse integer
Verse number
Example Request
curl https://api.prayerpulse.io/get-verse/YLT/1/1/1/
Example Response
{
  "pk": 1,
  "verse": 1,
  "text": "In the beginning of God's preparing the heavens and the earth --"
}
GET
/get-text/{translation}/{book}/{chapter}/
Retrieve all verses from a specific chapter.
Example Request
curl https://api.prayerpulse.io/get-text/YLT/43/3/
Example Response
[
  {
    "pk": 26118,
    "verse": 1,
    "text": "And there was a man of the Pharisees..."
  },
  // ... more verses
]
GET
/get-random-verse/{translation}/
Get a random Bible verse from the specified translation.
Example Request
curl https://api.prayerpulse.io/get-random-verse/YLT/

Search

Search for verses containing specific keywords or phrases.

GET
/search/{translation}/{query}/
Search for verses containing the specified text.
Path Parameters
translation string
Translation code
query string
Search term or phrase
Example Request
curl https://api.prayerpulse.io/search/YLT/love/
Example Response
{
  "verses": [
    {
      "pk": 26133,
      "book": 43,
      "chapter": 3,
      "verse": 16,
      "text": "for God did so love the world..."
    }
  ]
}

Books & Metadata

Retrieve book lists, verse counts, and translation metadata.

GET
/get-books/{translation}/
Get a list of all books in the specified translation.
Example Response
[
  {
    "bookid": 1,
    "name": "Genesis",
    "chapters": 50,
    "chronorder": 1
  },
  // ... 65 more books
]

User Features

Authenticated endpoints for user bookmarks, notes, and reading history.

Authentication Required: These endpoints require user authentication via Django session or OAuth.
POST
/save-bookmarks/
Save user bookmarks for specific verses.
GET
/get-bookmarks/{translation}/{book}/{chapter}/
Retrieve user bookmarks for a specific chapter.

Error Codes

Standard HTTP response codes are used:

Code Status Description
200 OK Request successful
400 Bad Request Invalid parameters or malformed request
404 Not Found Resource does not exist
500 Internal Server Error Server encountered an error

Best Practices

Performance Tips:
  • Use specific verse endpoints instead of downloading entire translations
  • Cache responses on your client to reduce API calls
  • Implement proper error handling for network failures
  • Always use HTTPS for secure connections
